{% include '파일명.html' %}: 지정된 HTML 파일을 현재 파일에 포함시킵니다.

19_Include 속의 코드의 내용을 확인해보면 include를 사용해서 같은 헤더를 여러 페이지에 사용해야할 경우 해당 태그를 사용해서
헤더의 내용을 한 번만 변경하면 모든 페이지에 반영이 됩니다.

그럼 여기서 의문은 이전의 ,Block과 include으 차이점이 무엇인지? 입니다.
둘다 여러 페이지에 재사용이 필요한 경우 사용되는데 의도와 동작 방식에 차이점이 있었습니다.

1. include:
    - 역할: 단순히 다른 HTML 파일의 내용을 그대로 복붙
    - 비유: 옆에 있는 펜을 그대로 사용하는것
    - 언제 사용하면 좋을까?:
        1. 네비게이션바
        2. 공용푸터
        3. 반복적으로 들어가는 코드
    
    - 특징:
        - 포함된 파일 안에서 또 include나 변수 사용이 가능합니다.
        - 상속 관계 필요없음 -> 어디서든 불러올 수 있습니다.

2. block, extends
    - 역할: 템플릿 상속 구조에서 해당 내용을 자식 템플릿에 채워 넣을 수 있게 예약을 한다입니다.
    - 비유: “빈 칸이 있는 시험지” — 시험지는 부모 템플릿이 만들고, 답을 쓰는 건 자식 템플릿
    - 언제 사용하면 좋을까?: 페이지 기본 틀(header, footer, 레이아웃)을 잡아두고, 각 페이지마다 다른 내용만 채울 때
    - 특징:
        - 반드시 extends와 함께 사용
        - 자식 템플릿이 block 이름을 지정해 내용을 덮어씁니다.


한줄로 정리
- include = “공용 조각 복사”
- block = “틀 속 빈칸 채우기”
