웹 애플리케이션에서의 요청은  클라이언트가 서버로부터 정보를 얻거나 서버에 정보를 전달하기 위해 보내는 HTTP 메시지입니다.


요청은 여러 데이터 전송 방법이 있는데 그중에 쿼리 매개변수[Query Parameter]와 요처으 바디[Request Body]가 흔히
사용됩니다.
웹 URL에서는 ?기호 뒤에오는 키-값 쌍으로 이루어진 문자열입니다.


## Query Parameter
FastAPI에서 Query클래스를 사용하여 쿼리 매개변수를 선언하고 검증 규칙을 추가할 수 있습니다.

Query클래스의 주요 옵션
Query클래스는 단순히 값을 전달하는 기능만 있는것이 아닙니다. 쿼리매개변수에 추가적인 메타데이터와 제약조건을 지정할 수 있습니다.

Query(default, OptionList)
* default: 매개변수의 기본값을 지정합니다. 값이 None이면 매개변수는 선택적이 됩니다.
    [옵션리스트중 필요한 설정을 넣을 수 있습니다.]
    - min_length: 문자열 매개변수에 대한 최소 길이를 지정합니다.
    - max_length: 문자열 매개변수에 대한 최대 길이를 지정합니다.
    - alias: 매개변수의 별칭을 지정합니다. 이를 통해 URL에서 사용하는 이름과 함수 내에서 사용하는 이름을 다르게할 수 있습니다.
    - deprecated: 매개변수가 더 이상 사용되지 않음을 명시합니다. API 문서에 표시되어 사용자가 해당 매개변수를 사용하지 않도록 경고합니다.
    - discription: 매개변수에 대한 설명을 추가합니다. API문서에 표시되어 매개변수의 사용목적이나 기대되는 값 등을 설명할 수 있습니다.
    - ge[greater or equal to]: 매개변수 값이 지정된 값 이상이여야함을 명시합니다.
    - le[less then equal to]: 매개변수 값이 지정된 값 이하여야함을 명시합니다.
    - regex:매개변수 값이 일치해야 하는 정규 표형식 패턴을 지정합니다.
    - title: 매개변수의 설명 제목을 지정합니다. 주로 API 문서에서 매개변수를 설명할 때 사용됩니다.
    - example: 매개변수의 예시 값을 제공합니다. 문서에서 매개변수의 예상 입력값을 보여주는데 도움을 줍니다.


## alias
alias 옵션은 클라이언트가 URL에 입력하는 쿼리 매개변수의 이름과 서버 내부에서 사용하는 변수명을 분리하여 설정할 수 있게 해줍니다.
이 기능은 내부 구현을 숨기거나 클라이언트에게 보다 친숙한 또는 직관적인 매개변수 이름을 제공하고자 할 때 유용합니다.
